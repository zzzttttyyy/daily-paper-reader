## 摘要
DiT视频生成由于迭代的全帧去噪而延迟严重，而先前的云边方法主要依赖于静态的步间解耦，无法利用帧间相似性或适应系统动态。我们提出了EcoVideo，一种用于动态帧间解耦的熵引导框架：早期自注意力熵提供了无需训练的帧级信息密度估计用于帧选择；云端大模型对稀疏的高熵关键帧进行去噪；边缘轻量级模型通过带有细化的运动感知插值重建剩余帧，以保证时间稳定性。EcoVideo还根据实时带宽和计算可用性调整关键帧预算和边缘细化深度，优化约束下的端到端延迟。在代表性的DiT视频生成器上的实验表明，在低带宽、计算受限的边缘设置中，改进了质量-效率权衡，并实现了高达2.9倍的端到端加速。代码可在https://github.com/IF-LAB-PKU/EcoVideo获取。

## Abstract
DiT video generation is latency-intensive due to iterative full-frame denoising, while prior cloud-edge methods largely rely on static inter-step decoupling and cannot leverage inter-frame similarity or adapt to system dynamics. We propose EcoVideo, an entropy-orchestrated framework for dynamic inter-frame decoupling: early-stage self-attention entropy provides a training-free estimate of frame-wise information density for frame selection; a cloud large model denoises sparse high-entropy keyframes; and an edge lightweight model reconstructs the remaining frames via motion-aware interpolation with refinement for temporal stability. EcoVideo further adapts the keyframe budget and edge refinement depth to real-time bandwidth and compute availability, optimizing end-to-end latency under constraints. Experiments on representative DiT video generators show improved quality--efficiency trade-offs and up to 2.9x end-to-end speedup in low-bandwidth, compute-limited edge settings. Code is available at https://github.com/IF-LAB-PKU/EcoVideo.

## 1. 核心问题与整体含义
- **研究动机**：Diffusion Transformer（DiT）视频生成依赖逐步的全帧去噪，计算延迟极高，难以大规模部署。现有云边协作方法（如HybridSD、EC-Diff）采用**静态步间解耦**（大模型做早期去噪、小模型做后期去噪），存在两个根本缺陷：（1）忽略帧间相似性，对高度相似的帧仍重复计算；（2）无法适应动态变化的网络带宽和边缘计算资源，导致实际端到端效率不稳定甚至更差。
- **整体含义**：需要一种能动态利用帧间信息稀疏性、自适应系统变化的云边协作范式，以在保持生成质量的同时显著加速。

## 2. 方法论
### 核心思想
- 将云边协作从**步间解耦**（step-level decoupling）转变为**帧间解耦**（frame-level decoupling）：云端大模型仅生成信息密度高的关键帧，边缘轻量模型通过插值补全其余帧，并根据实时系统状态动态调整关键帧预算和边缘细化深度。

### 关键技术细节
1. **帧间注意力熵分析（Inter-frame Attention Entropy Analysis）**：
   - 在早期扩散步骤（前10%作为预热阶段）计算每个self-attention token的熵，按帧聚合得到帧级熵 $e_f$。
   - 高熵 → 高信息密度（复杂运动、结构变化）；低熵 → 冗余或相似帧。
   - 通过指数移动平均（EMA）稳定估计，作为后续关键帧选择的无需训练的信号。

2. **帧级熵引导生成（Frame-level Entropy-Orchestrated Generation）**：
   - **关键帧选择**：强制包含首帧和末帧，其余 $K-2$ 个关键帧按熵排名选取；上下文注入：将非关键帧的潜变量作为冻结的上下文token，辅助云端去噪时保持全局时间结构。

3. **云边动态感知自适应（Cloud-Edge Dynamics-aware Decoupling Adaptation）**：
