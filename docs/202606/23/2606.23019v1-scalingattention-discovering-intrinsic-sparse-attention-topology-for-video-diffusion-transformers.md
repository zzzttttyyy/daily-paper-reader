## 摘要
虽然扩散变换器（DiTs）已经革新了高保真视频生成，但它们对3D全注意力的依赖造成了二次计算瓶颈。现有的稀疏方法面临两难困境：动态剪枝存在过高的运行时开销和内存碎片化，而静态启发式方法无法捕捉细粒度依赖关系。在这项工作中，我们提出了ScalingAttention，一个基于关键归纳偏置的无训练框架：虽然单个激活是输入相关的，但每个头的高质量注意力区域会迅速收敛到一个稳定的、与提示无关的内在稀疏拓扑。该拓扑是权重编码的、尺度不变的且易于提取。ScalingAttention通过以下方式将拓扑发现与稀疏控制解耦：（1）WEST（权重编码稀疏拓扑），离线提取鲁棒的块稀疏先验掩码以消除运行时搜索；（2）FAST（保真度感知灵敏度调优），基于扩散保真度要求自适应调节头级稀疏度。为确保实际加速，我们协同设计了硬件对齐的逐位块稀疏内核。在Wan2.1上的实验显示，端到端加速高达1.90倍，同时保持卓越保真度，在现有最佳基线之上建立了新的帕累托前沿。

## Abstract
While Diffusion Transformers (DiTs) have revolutionized high-fidelity video generation, their reliance on 3D full attention creates a quadratic computational bottleneck. Existing sparse methods face a dilemma: dynamic pruning suffers from prohibitive runtime overhead and memory fragmentation, while static heuristics fail to capture fine-grained dependencies. In this work, we propose ScalingAttention, a training-free framework grounded in a key inductive bias: while individual activations are input-dependent, the high-mass attention regions for each head rapidly converge to a stable, prompt-agnostic Intrinsic Sparse Topology. This topology is weight-encoded, scale-invariant, and efficient to extract. ScalingAttention decouples topology discovery from sparsity control via: (1) WEST (Weight-Encoded Sparse Topology), which extracts a robust block-sparse prior mask offline to eliminate runtime search; (2) FAST (Fidelity-Aware Sensitivity Tuning), which adaptively tunes head-wise sparsity based on diffusion fidelity requirements. To ensure practical acceleration, we co-design a hardware-aligned bit-wise block-sparse kernel. Experiments on Wan2.1 show up to 1.90X end-to-end speedup with superior fidelity, establishing a new Pareto frontier over state-of-the-art baselines.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：视频扩散变换器（Video Diffusion Transformers，如Wan2.1、HunyuanVideo）依赖于3D全注意力，计算复杂度为 \(O(N^2)\)，导致推理极其昂贵（如Wan2.1-14B生成5秒视频需近30分钟）。现有稀疏注意力方法存在两难：**动态方法**（如SVG2）虽能捕捉输入依赖的稀疏模式，但运行时开销大、内存碎片化严重，难以利用Tensor Core；**静态方法**（如Radial、PARO）使用固定启发式模式（滑动窗口、膨胀模式），虽硬件友好，但无法适应语义结构，导致生成质量下降。
- **整体含义**：论文提出一个关键归纳偏置——视频DiT中每个注意力头的**高质注意力区域（high-mass attention regions）** 会迅速收敛到**与提示无关的稳定内在稀疏拓扑（Intrinsic Sparse Topology）**，该拓扑由权重编码、尺度不变，且易于提取。基于此，作者设计了**无需训练的ScalingAttention框架**，将稀疏注意力问题解耦为：**拓扑发现（在哪里允许注意力）** 和**敏感度控制（能容忍多少稀疏度）**，从而同时实现高保真度和高效率。

## 2. 方法论：核心思想、关键技术细节

**核心思想**：利用视频DiT中注意力头固有的权重编码稀疏拓扑，将拓扑发现与稀疏度控制解耦，通过离线构建静态掩码和动态保真度感知调整，实现无运行时搜索的高效稀疏注意力。

**关键技术细节**：

- **WEST（Weight-Encoded Sparse Topology）**：
  - 使用校准集（约27个提示）为每个头计算注意力矩阵，按 \(B \times B\) 块（\(B=128\)）聚合得到块显著性矩阵 \(S^{(u,v)}_i = \frac{1}{B^2}\sum_{p\in B_u}\sum_{q\in B_v} A^{(p,q)}_i\)。
  - 构造**阈值图（Threshold Map）** \(T^{(u,v)} = \min\{p \mid \exists i, (u,v) \in \text{Top-k}(S_i, p)\}\)，记录每个块被激活的最小稀疏度阈值。这编码了完整的稀疏层次，可在推理时通过简单的比较 \(M_p = I(T \leq p)\) 快速生成任意目标密度下的二进制掩码。
  - 该拓扑是**尺度不变**的：从480p代理分辨率提取的掩码与原始720p掩码的召回率>96%，可复用。

- **FAST（Fidelity-Aware Sparsity Tuning）**：
  - 使用**Hellinger距离**作为保真度指标（克服余弦相似度过早饱和的问题）：\(H(P,Q) = \frac{1}{\sqrt{2}}\sqrt{\sum_{i}(\sqrt{p_i}-\sqrt{q_i})^2}\)，定义**保真度分数** \(F(P,\tilde{P}) = 1 - H(P,\tilde{P})\)。
  - 定义一个**参数化目标保真度函数** \(A(l,t) = 1 - [\beta(1 - \frac{l+\omega t}{N_L + \omega N_T})^\gamma + (1-\beta)]\)，用于为每个层\(l\)和时间步\(t\)设定可容忍的保真度下限（早期步骤和浅层要求更高保真度）。
  - 通过离线预计算每个头在WEST拓扑下的**保真度-稀疏度曲线**，在推理前根据全局密度目标查表确定每个头的最优稀疏度阈值 \(p^*\)，生成静态掩码。
  - 为降低离线校准成本，采用**分层采样（Stratified Sampling）**：仅取约16行（0.05%的注意力矩阵）即可高精度近似Hellinger距离。

- **系统-算法协同设计（CRM Kernel）**：
  - **分辨率可缩放**：通过保守双线性插值（带取整规则）将基础分辨率下的阈值图缩放到任意目标分辨率，存储成本恒定。
  - **位块稀疏内核（crm kernel）**：采用**压缩行掩码（Compressed Row Mask, CRM）** 表示，每行使用对齐的uint32数组存储块激活状态（每位表示一个128×128块），通过硬件位操作（bit scan, bit clear）快速遍历活跃KV块，无缝替代FlashAttention中的密集块迭代器，支持Tensor Core高效计算。在密集设置下额外开销<10%，在50%稀疏度下延迟降低40-60%。

**算法流程**（Algorithm 1）：离线阶段包括（1）对每个头、每个校准样本计算块矩阵并更新阈值图；（2）计算保真度-密度曲线。在线阶段：给定全局密度目标，根据FAST确定每个头的阈值，从阈值图生成静态块掩码，供推理时使用。

## 6. 主要结论与发现

1. **内在稀疏拓扑的存在**：视频DiT中每个注意力头具有权重编码的静态稀疏拓扑（稳定、与提示无关、尺度不变、宽高比鲁棒），这是本文的核心发现。
2. **ScalingAttention实现高效加速**：在Wan2.1-1.3B上达到1.65×加速（50%密度），在Wan2.1-14B上达到1.90×加速（52.5%密度），在HunyuanVideo上达到1.73×加速（55%密度），同时PSNR等指标优于SVG和SVG2。

## 8. 不足与局限

- **粒度瓶颈（Granularity Bottleneck）**：块大小为128×128，在极端稀疏度（<40%）下无法保留细粒度关键token，导致性能下降。论文提到可结合token重排缓解，但未给出彻底解决方案。
- **静态包络冗余**：WEST提取的是所有激活区域的并集，对于局部运动场景会保留比动态oracle更多的token，存在冗余。
- **校准集依赖**：虽然只需少量提示，但校准提示的选择可能影响拓扑质量（尤其是在分布外数据上）。论文对跨域（如不同视频风格）的鲁棒性验证有限。
- **实验覆盖局限性**：仅在三个公开模型上测试，未涵盖更多视频DiT架构（如Open-Sora、CogVideoX）；未评估极端长视频（如数分钟）或超高分辨率下的表现；缺乏用户研究（主观评估）。
- **硬件依赖**：crm内核针对NVIDIA GPU（Tensor Core和位操作）设计，在非NVIDIA硬件上效率未知。
- **资源细节不透明**：未明确GPU型号和算力消耗的完整报告，仅以锁频描述，可复现性受限于环境信息不足。
