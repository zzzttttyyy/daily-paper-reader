## 摘要
文本引导的扩散模型已成为真实图像视觉编辑的有效工具，其中编辑后的图像必须遵循目标指令，同时保持与编辑无关的结构。大多数免训练编辑器依赖于反转：将源图像映射到噪声潜变量轨迹，并将终端潜变量重新用于目标提示去噪。这种重用有助于保持，但也将源重建与目标编辑耦合在一起。由此产生的轨迹不匹配可能损害背景/布局细节，或过度约束预期的编辑。本文提出了一致反转（Consistent-Inversion），一种用于结构保持视觉编辑的免训练反向一致性引导框架。一致反转不将反转后的源潜变量视为固定初始化，而是检查在源提示下，中间目标轨迹是否可以向源反转轨迹反向。为了使这种检查定义明确，我们构建了一个辅助的目标侧噪声表示，执行源引导的反向去噪，并将由此产生的反向一致性差异作为选定早期目标去噪步骤的校正信号。该方法不更新模型参数，与基于反转的编辑器兼容，并且在稀疏应用时仅引入较小的推理开销。在PIE-Bench上的实验表明，一致反转在统一的SD3.5协议下提高了背景和结构保真度，同时保持了目标提示对齐，兼容性实验进一步验证了在经典Stable-Diffusion反转流程上的相同校正原理。

## Abstract
Text-guided diffusion models have become effective tools for real-image visual editing, where the edited image must follow a target instruction while preserving editing-irrelevant structure. Most training-free editors rely on inversion: a source image is mapped to a noisy latent trajectory and the terminal latent is reused for target-prompt denoising. This reuse is useful for preservation, but it also couples source reconstruction and target editing. The resulting trajectory mismatch may either damage background/layout details or over-constrain the intended edit. This paper presents Consistent-Inversion, a training-free reverse consistency guidance framework for structure-preserving visual editing. Instead of treating the inverted source latent as a fixed initialization, Consistent-Inversion checks whether an intermediate target trajectory can be reversed toward the source inversion trajectory under the source prompt. To make this check well-defined, we construct an auxiliary target-side noise representation, perform source-guided reverse denoising, and use the resulting reverse consistency discrepancy as a correction signal for selected early target denoising steps. The method does not update model parameters, is compatible with inversion-based editors, and introduces only a small inference overhead when applied sparsely. Experiments on PIE-Bench show that Consistent-Inversion improves background and structural fidelity under a unified SD3.5 protocol while maintaining target-prompt alignment, and compatibility experiments further verify the same correction principle on classical Stable-Diffusion inversion pipelines.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：文本引导的扩散模型在真实图像编辑中，普遍采用“反转-去噪”框架（如DDIM反转）：将源图像通过源提示反转到一个噪声潜变量轨迹，再用该终端潜变量作为目标提示去噪的起点。这种复用虽然有助于保持编辑无关的结构，但**轨迹不匹配**问题突出——源重建与目标编辑共享同一潜变量初始状态，导致要么损伤背景/布局细节，要么过度约束预期的编辑。
- **背景**：现有免训练编辑器主要依靠注意力替换、特征注入、潜变量替换、提示优化等间接控制，缺乏对目标轨迹与源轨迹一致性的显式检查。这些方法虽能缓解问题，但本质上是启发式的，无法定量评估目标分支的结构漂移。
- **研究意义**：本文首次将轨迹层次的可逆性作为结构保持的判据，提出一种**反向一致性校正**机制，在不训练、不修改模型参数的前提下，通过源引导的反向去噪计算偏移量，纠正早期去噪步的结构漂移，从而在保持目标语义编辑的同时，显著提升背景和结构保真度。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程
- **核心思想**：一个结构可靠的编辑结果，其潜变量轨迹在源提示条件下应近似可逆。即，从目标分支的中间潜变量出发，先反向构造辅助目标侧噪声表示，再用源提示去噪回到对应时间步，所得潜变量应与源反转轨迹上的对应点相近。两者的差异即为**反向一致性偏移**，可作为校正信号注入目标分支。
- **关键技术细节**：
  1. **辅助目标侧噪声构造**：对目标分支当前状态 \(z^{+}_k\)，通过DDIM反转（用目标提示）得到 \(\hat{z}^{+}_T\)，模拟一个“目标端的终端噪音”。
  2. **源引导反向去噪**：从 \(\hat{z}^{+}_T\) 出发，使用**源提示**进行DDIM采样回到时间步 \(k\)，得到 \(z^{*}_k\)。
  3. **偏移计算与校正**：偏移量 \(O^{-}_k = z^{-}_k - z^{*}_k\)，其中 \(z^{-}_k\) 是源反转轨迹上的对应潜变量；校正公式：\(z^{+}_k \leftarrow z^{+}_k + \lambda_k O^{-}_k\)，\(\lambda_k\) 为缩放系数。
- **高效实现**：全文应用仅在**早期少数时间步**（如 \(k = T-1, T-3, T-5\) 等）进行校正，因为全局布局在早期去噪阶段确定，后期主要细化纹理。默认使用3个校正步，每个步额外增加一次反转和一次去噪，总额外计算约2m次网络评估（m为校正步数），开销较小。
- **算法流程**（伪代码形式）：
  - 1) 用DDIM反转源图像 \(z^{-}_0\) 得到 \(z^{-}_1, z^{-}_2, ..., z^{-}_T\)（源提示S）。
  - 2) 初始化目标分支 \(z^{+}_T = z^{-}_T\)。
  - 3) 对 \(t = T\) 到 1：
     - 执行目标提示T引导的一次去噪，得到 \(z^{+}_{t-1}\)。
     - 如果 \(t-1 \in \mathcal{K}\)（校正集），则：
       - 构建 \(\hat{z}^{+}_T = \text{Inv}_{t-1 \to T}(z^{+}_{t-1}, T)\)（目标提示反转）。
       - 计算 \(z^{*}_{t-1} = \text{Sam}_{T \to t-1}(\hat{z}^{+}_T, S)\)（源提示去噪）。
       - 计算偏移 \(O^{-}_{t-1} = z^{-}_{t-1} - z^{*}_{t-1}\)。
       - 更新 \(z^{+}_{t-1} \leftarrow z^{+}_{t-1} + \lambda_{t-1} O^{-}_{t-1}\)。
  - 4) 输出编辑潜变量 \(z^{+}_0\)。

- **关键发现**：
  - 该校正主要作用于早期去噪步，通过改变源兼容的轨迹分量，后续步骤继承优化后的几何。

## 8. 不足与局限
- **适用范围假设**：方法假设源提示和目标提示共享场景布局，主要差别在物体类别、纹理、颜色、材质、风格等。对于涉及大幅度视角变化、目标数量变化、全局空间重排等编辑，过于强烈的校正可能抑制预期变化。作者建议在这些情况下使用更保守的缩放和稀疏校正。
- **评估局限**：仅使用PIE-Bench一个基准，未涉及其他编辑数据集或真实用户偏好研究；缺乏对更复杂指令（如多目标编辑、区域约束）的测试。
- **计算开销**：虽稀疏校正，但每校正一步需额外两次网络评估（反转和目标侧反转+源去噪），在时间步较多且校正集较大时，推理时间可能增加一倍以上。目前默认3步增加约0.2s/图，可接受。
- **未考虑编辑掩码**：文中提及可与掩码结合，但实验未采用，因此校正可能对已编辑区域也施加一定保持，导致编辑幅度减弱（CLIP-T微降0.1%）。
- **尚未与最新Flow-based或一步编辑方法对比**：文中提到RF Inversion、FlashEdit等，但仅作为背景，未在实验部分纳入对比。随着这些新方法的发展，Consistent-Inversion的适配性与优势尚需进一步验证。
