## 摘要
视频扩散变换器处理长时空序列，使得自注意力成为高分辨率视频生成的主要瓶颈。免训练稀疏注意力降低了这一成本，但自适应Top-p路由在多GPU序列并行下会产生不均匀的每头工作负载。由此产生的工作负载异质性使稀疏注意力变成了一个等级级别的落后任务问题。我们提出了FVAttn，一个免训练稀疏注意力系统，它提高了多GPU序列并行下自适应稀疏注意力的分布式执行效率。FVAttn使用Top-p路由、Top-k安全底线和视频感知块组织作为稀疏路由前端，然后在运行时修复物化掩码。运行时负载平衡通过点对点通信迁移少数重头，以缩短当前关键路径。感知空闲的稀疏增强用额外的高价值块填充剩余的非关键等级的闲置，而重叠将调度和迁移开销隐藏在现有计算之后。在步进蒸馏的Wan2.2 I2V上，FVAttn将平均负载不平衡从1.34降低到1.08，并相比FlashAttention实现了$4.41\times$的注意力加速，同时实现了$2.02$--$2.11\times$的DiT推理加速，且视频质量具有竞争力。

## Abstract
Video Diffusion Transformers process long spatio-temporal sequences, making self-attention the main bottleneck in high-resolution video generation. Training-free sparse attention reduces this cost, but adaptive Top-$p$ routing creates uneven per-head workloads under multi-GPU sequence parallelism. The resulting workload heterogeneity turns sparse attention into a rank-level straggler problem. We present \method{}, a training-free sparse-attention system that improves the distributed execution efficiency of adaptive sparse attention under multi-GPU sequence parallelism. \method{} uses Top-$p$ routing, a Top-$k$ safety floor, and video-aware block organization as the sparse-routing frontend, then repairs the materialized mask at runtime. Runtime Load Balancing migrates a small number of heavy heads via P2P communication to shorten the current critical path. Slack-Aware Sparse Augmentation fills residual non-critical-rank slack with additional high-value blocks, while overlap hides scheduling and migration overhead behind existing computation. On step-distilled Wan2.2 I2V, \method{} reduces average load imbalance from 1.34 to 1.08 and delivers a $4.41\times$ attention speedup over FlashAttention, while achieving a $2.02$--$2.11\times$ DiT inference speedup with competitive video quality.

# 论文总结：FVAttn: Adaptive Sparse Attention with Runtime Load Balancing for Video Generation

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：视频扩散Transformer（Video DiTs）在高分辨率视频生成中面临严重推理延迟瓶颈，其中自注意力（self-attention）占据了总推理时间的74.1%（以Wan2.2-14B I2V为例）。虽然步进蒸馏（step-distillation）可将去噪步数减少至4步，但每步的序列长度和二次注意力成本不变，注意力仍是主要瓶颈。
- **现存问题**：免训练稀疏注意力可降低计算成本，但自适应Top-p路由会产生每头（per-head）工作负载不均匀。在多GPU序列并行（如Ulysses风格）下，这种不均匀导致某些GPU成为拖累者（stragglers），其他GPU等待，抵消稀疏注意力带来的理论加速。
- **核心挑战**：如何在保留自适应稀疏注意力保真度优势的同时，实现多GPU下的负载均衡和高效分布式执行。

## 2. 论文提出的方法论

### 核心思想
FVAttn是一个免训练稀疏注意力系统，通过**运行时负载均衡（RLB）**和**松弛感知稀疏增强（SASA）**，在不增加全局关键路径的前提下修复和利用工作负载不均。

### 关键技术细节

1. **稀疏路由前端**：
   - 使用**Top-p路由**自适应选择关键块（key blocks），并加入**Top-k安全下限**防止过度稀疏化。
   - 采用**视频感知块组织**（基于希尔伯特曲线重排块），使相邻token落入相近块中。
   - 路由阶段同时产生每头工作量`L_h`和块重要性排序，供后续RLB和SASA使用。

2. **运行时负载均衡（RLB）**：
   - **动机**：负载不均匀高度集中在少数“重头”上，迁移少量头即可大幅降低最大负载。
   - **机制**：稀疏掩码物化后，RLB基于实际观测到的每头工作量，通过**受限的点对点（P2P）通信**迁移重头。默认8-GPU设置下，每个GPU最多迁移一个本地头（约20%），形成置换σ。
   - **搜索**：所有GPU通过一次轻量级all-gather获取全局密度信息，独立运行确定性搜索，选择使最大秩负载`max L_r`最小化（平局时最小化方差）的迁移计划。搜索开销在数百微秒内。

3. **松弛感知稀疏增强（SASA）**：
   - **动机**：RLB后非关键GPU仍有剩余松弛（slack），等待慢GPU完成，这些空闲计算能力可被利用。
   - **机制**：对每个具有足够松弛的秩`r`，分配额外稀疏预算`B_r = η₁·ΔL_r`（η₁=0.8为增强系数，ΔL_r为松弛量）。SASA根据之前路由已排序的块重要性顺序，添加额外高价值块到掩码中，且不超出松弛，避免创建新拖累者。
   - **触发条件**：仅当负载不平衡因子ρ≥k₁（k₁=1.05）且松弛比例超过η₂=0.07时启用SASA；ρ≥k₂（k₂=1.10）时先启用RLB再启用SASA。

4. **高效实现（重叠优化）**：
   - **CPU-GPU重叠**：CPU搜索RLB计划和计算SASA预算与GPU的V量化重叠。
   - **计算-通信重叠**：P2P头迁移异步发起，GPU同时处理未迁移头的V量化和SASA增强，仅在消费迁移状态时同步。
   - **额外同步优化**：移除不必要的CPU-GPU同步，流式化管理。

### 算法流程（文字说明）
1. 序列到头的All-to-All后，每个GPU获取完整序列的子集头。
2. 执行自适应稀疏路由（Top-p + Top-k下限），生成稀疏掩码，同时得到每头工作量`L_h`和块重要性排序。
3. 根据ρ值决定启用机制：
   - 若ρ < 1.05：跳过额外机制。
   - 若1.05 ≤ ρ < 1.10：仅启用SASA。
   - 若ρ ≥ 1.10：先执行RLB（迁移重头，更新头到秩映射），再对非关键秩执行SASA。
4. 执行块稀疏注意力（block-sparse attention）。
5. 执行反向All-to-All恢复头顺序。

## 6. 主要结论与发现

- FVAttn在三个工作负载上一致改进了质量-效率Pareto前沿。
- **I2V上**：负载不平衡因子从1.34降至1.08，注意力相比FlashAttention加速4.41×，DiT推理加速2.02–2.11×，且质量（VBench、PSNR等）优于或等同于竞争方法。
- **Animate上**：完整FVAttn（Top-p=0.95）比FVAttn-Base减少7.4% DiT延迟，同时PSNR提升0.458 dB。
- **RLB vs. db-SP**：RLB（基于当前掩码的修复）比db-SP（基于历史布局）更快（15.18s vs. 15.51s on Animate），且ρ更低（1.08 vs. 1.22 on I2V），验证了“当前工作负载修复”优于“历史复用”的假设。
- **SASA**：在不增加关键路径的前提下提升掩码覆盖率，改善质量指标。
- **重叠优化**：额外运行时开销仅占最终注意力延迟的1.87%（约0.7ms）。

## 7. 优点

- **创新点鲜明**：识别了“自适应稀疏性在并行系统中产生拖累者”这一实际问题，并提出轻量级运行时修复方案。
- **方法实用**：RLB和SASA基于物化后的真实工作负载，避免了预测的不准确性；受限迁移策略（最多移一个头）确保通信开销极小。
- **全面消融**：逐步验证每个模块的单独贡献，并量化开销分解。
- **高效率实现**：通过CPU-GPU和计算-通信重叠隐藏大部分开销。
- **通用性**：RLB+SASA可作为即插即用组件应用于其他Top-p稀疏注意力系统（如SpargeAttention）。

## 8. 不足与局限

- **适用范围**：主要针对长时空序列视频生成；对短序列或稀疏性弱的任务（如图像生成），加速和负载均衡收益会下降。
- **硬件依赖性**：P2P迁移和重叠优化依赖于GPU间高速互连（NVLink）。在PCIe连接或通信带宽较低的设备上，收益可能受限，需要重新校准参数（迁移预算、触发阈值等）。
- **并行范式局限**：仅验证了Ulysses风格序列并行；未来需探索Ring Attention、USP等其它范式。
- **未验证训练场景**：方法设计用于推理阶段的免训练稀疏注意力；训练时稀疏注意力的适用性待探索。
